# NotionClip Master Execution Note

Date: April 1, 2026
Owner: Product + Founding Team

## 1) What We Are Building
NotionClip is not a generic summarizer.
It is an intent-first intelligence layer between content consumption and actual knowledge, with structured output saved to Notion.

Core promise:
From long-form content to output-ready intelligence in under 60 seconds.

Brand anchor:
Watch less. Know more.

## 2) The Real Risk Right Now
Main risk is not feature count.
Main risk is first-use clarity.

If a first-time user cannot get a meaningful win in the first 60 seconds, advanced depth does not matter.

## 3) Ideal First 60-Second Loop (Default Path)
1. User clicks Try Free.
2. User lands in app with YouTube preselected.
3. Study mode is preselected.
4. One clear field: Paste YouTube URL.
5. One clear CTA: Generate timestamped study notes.
6. Right panel explains what will be produced while processing.
7. Output appears with clickable timestamp moments.
8. Left panel auto-collapses so output becomes primary.
9. User saves or exports.

Rule:
Advanced controls are hidden by default and revealed only after first output.

## 4) Product Surface Strategy
### Homepage role
Convert interest into one action.

Homepage should communicate only:
1. Who this is for.
2. What result appears.
3. How fast it appears.
4. One CTA path.

### App page role
Deliver first value fast, then expose power.

App structure:
1. Fast path lane: URL + mode + process.
2. Guided right panel before output.
3. Output-first workspace after processing.
4. Advanced controls disclosed progressively.

### Library role
Retention and reuse.

Library should answer:
1. What did I learn?
2. Where is that moment in source content?
3. What should I review next?

## 5) Competitive Truth (Snipo, NotebookLM, NoteGPT)
### Where they are strong
1. Lower initial friction.
2. Browser-native behavior (especially Snipo).
3. Strong post-processing interaction loops.

### Where we can win
1. Notion-native structured output quality.
2. Intent-first extraction (question-first behavior).
3. Multi-source synthesis for real learning outcomes.
4. Timestamped study moments linked to source and Notion structure.

## 6) Biggest Gaps To Close
1. Timestamp notes must be first-class and visible in core output.
2. First-use needs one obvious path, not many decisions.
3. Post-output interaction needs stronger learning habit loop.
4. Distribution friction remains high without extension workflow.

## 7) Timestamp Notes Feature Blueprint (Priority Feature)
Goal:
Match and surpass timestamp utility expectations while keeping NotionClip differentiated.

### User-facing behavior
1. Each key insight includes source timestamp.
2. Clicking a timestamp opens source at exact moment.
3. Notion export preserves timestamp links and hierarchy.
4. Moments are grouped by concept and mode output section.

### Product requirements
1. Timestamp chips visible in right-panel output cards.
2. Jump-to-source interaction from each chip.
3. Timestamp sorting and dedupe for repeated concepts.
4. "Save key moments" section in final export.

### Notion export requirements
1. Include timestamp links as callouts or bullets under each key point.
2. Preserve source title and section context.
3. Keep readability high (no transcript dump blocks).

### Success metric
1. Percentage of outputs containing timestamp links.
2. Timestamp click-through rate.
3. Save-to-Notion rate for timestamped outputs.

## 10) Funnel Metrics We Must Track Weekly
1. Landing to Try Free click-through.
2. Try Free to first process click.
3. Process click to first output shown.
4. Median time-to-first-value.
5. Output to save/export conversion.
6. D1 and D7 return rate.
7. Timestamp interaction rate.

## 11) Decision Rules
1. If first-output conversion is low, simplify onboarding further before adding features.
2. If output shown but save rate is low, improve output usefulness and export formatting.
3. If users activate but do not return, prioritize retention features over new ingestion modes.
